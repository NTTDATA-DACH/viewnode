## Codebase Patterns
- For Cobra subcommands that intentionally reuse a shorthand owned by a root flag, register the shorthand on the leaf command and verify it through a small root command tree with `Execute()` so parsing matches the real CLI path.
- For Cobra leaf flags backed by package-level variables, only honor the flag value when that command's flag was actually set; otherwise shared state can leak between cloned commands and tests.
- Normalize optional string filter flags before applying match logic so an explicitly empty value behaves the same as omitting the flag.
- Keep filter-aware empty-state messaging inside `srv/view.go` and pass only the minimal CLI filter context on `srv.ViewNodeData` instead of branching the root command print path.
- For root empty-state output regressions, test `cmd.executePrintOut` with a `ViewNodeData` value that still includes the unscheduled-pod placeholder at `Nodes[0]`; the root path does not emit an actually empty slice.
