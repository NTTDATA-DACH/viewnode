# Ralph Progress Log
## Codebase Patterns
- Top-level Cobra commands without subcommands render under `Additional help topics` in root help output, so help assertions should match that section instead of `Available Commands`.
- Commands that honor the global `--kubeconfig` flag must pass the root Cobra command into `config.Initialize`; using the leaf subcommand directly can miss the root-level flag lookup.
- Cobra tests that need inherited persistent flags should execute through a root command tree with `Execute()`; calling a leaf command's `RunE` directly skips the normal flag propagation.
- `README.md` command listings and CLI usage examples are maintained manually, so user-facing CLI command changes should update the README in the same change.
- Cobra command groups in `cmd/<name>` follow the `ctx` pattern: define the group command in `<name>.go`, keep each subcommand in its own file, and register them together from the package `init()`.
